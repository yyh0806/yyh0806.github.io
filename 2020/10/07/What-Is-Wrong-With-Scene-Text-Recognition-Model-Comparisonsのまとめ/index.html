What Is Wrong With Scene Text Recognition Model Comparisons? Dataset and Model Analysis

背景

Many new proposals for scene text recognition (STR) models have been introduced in recent years. While each claim to have pushed the boundary of the technology, a holistic and fair comparison has been largely missing in the field due to the inconsistent choices of training and evaluation datasets. This paper addresses this difficulty with three major contributions. First, we examine the inconsistencies of training and evaluation datasets, and the performance gap results from inconsistencies. Second, we introduce a unified four-stage STR framework that most existing STR models fit into. Using this framework allows for the extensive evaluation of previously proposed STR modules and the discovery of previously unexplored module combinations. Third, we analyze the module-wise contributions to performance in terms of accuracy, speed, and memory demand, under one consistent set of training and evaluation datasets. Such analyses clean up the hindrance on the current comparisons to understand the performance gain of the existing modules. Our code is publicly available1 .

近年、シーンテキスト認識（STR）モデルに関する多くの新しい提案が紹介されています。それぞれがテクノロジーの限界を押し広げたと主張していますが、トレーニングと評価データセットの選択が一貫していないため、この分野では全体的かつ公正な比較がほとんど見られません。

解決する課題

トレーニングと評価データセットの選択が一貫していないため、この分野では全体的かつ公正な比較がほとんど見られません。

解決策

一貫したトレーニングおよび評価データセットを前提として、統合された4段階のSTRフレームワークを開発し、精度、速度、およびメモリ要件の観点から、以前に提案されたモデルのパフォーマンスを広範囲に評価します。

STRフレームワーク

  1. 変換(Trans) : Spatial Transformer Network(STN)を使って、入力画像を正規化する。
  2. 特徴抽出(Feat) : 入力画像と文字の関連する属性をマップし、無関係なもの（フォント、色、サイズ、背景など）を抑制する。
  3. シーケンスモデリング(Seq) : シーケンス全体の予測は、1つの文字の予測よりも正確である。
  4. 予測(Pred) : 抽出された特徴から文字を予測する。
